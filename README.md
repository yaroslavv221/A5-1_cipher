# A5/1_cipher

Клиент-серверное приложение для защищенной односторонней голосовой связи. Проект реализует захват звука, его потоковое шифрование алгоритмом A5/1 "на лету" и передачу по сети с удаленным управлением ключами через Telegram.

## Стек технологий

### **Стек технологий:** 

- Python 3.10+
- KivyMD (GUI)
- PyAudio (PortAudio)
- pyTelegramBotAPI. 

### **Сеть и Архитектура:** 

- Raw TCP Sockets
- Multithreading
- Client-Server
- IPC (Subprocess/Pipes). 

### **Аудио и Безопасность:** 
ALSA (aplay)
Raw PCM (8kHz/16-bit)


### **ОС:** 

- Windows (Client),
- Astra Linux (Server).

---

## Ключевые возможности

* **Кастомная криптография:** Программная реализация потокового шифра A5/1 без сторонних крипто-библиотек.
* **Стриминг аудио:** Захват сырого PCM-аудио и передача по сырым TCP-сокетам без задержек.

* **Современный GUI:** Десктопный клиент на базе Kivy/KivyMD.

* **Telegram-управление:** Генерация и ротация сессионных ключей на сервере через команды бота.

* **Нативная интеграция:** Прямой проброс расшифрованного аудиопотока в аудиоподсистему Linux (ALSA) через `aplay`.

## Установка и запуск

Проект разделен на две независимые части. Для тестирования их можно запустить как на разных машинах в локальной сети, так и на одном ПК.

### 1. Подготовка (Для обеих машин)

Клонируйте репозиторий:


- git clone https://github.com/yaroslavv221/A5-1_cipher

- cd A5-1_cipher

### 2. Установка необходимых библиотек

```
pip install -r requirements.txt
```

### 2. Запуск Сервера (Astra Linux)

Сервер принимает подключения, раздает ключи и воспроизводит звук.

1. Убедитесь, что в системе установлена утилита `aplay` (обычно встроена в ядро Linux).

2. Установите зависимости:

    ```
    pip install pyTelegramBotAPI
    ```
    
3. Замените `BOT_TOKEN` в файле `server/astra_server.py` на токен вашего бота.

4. Запустите сервер:


    ```
    python3 server/astra_server.py
    ```

_В консоли отобразится ваш локальный IP-адрес, который нужно будет ввести в клиенте._


### 3. Запуск Клиента (Windows PC)

Клиент захватывает звук с микрофона, шифрует его и отправляет на сервер.

1. Установите зависимости (потребуется установленный C++ компилятор для PyAudio):


    ```
    pip install -r requirements.txt
    ```

2. Запустите графический интерфейс:

```
    python client/desktop_app.py
```

## Инструкция по использованию

1. **Генерация ключа:** Напишите вашему Telegram-боту команду `/generate`. Сервер создаст новый безопасный ключ.

2. **Настройка клиента:** В открывшемся окне клиента введите IP-адрес сервера (указан в консоли сервера при запуске).

3. **Синхронизация:** Нажмите кнопку `СИНХРОНИЗИРОВАТЬ КЛЮЧ`. Клиент подключится к порту `9091` и скачает текущий сессионный ключ.

4. **Связь:** Нажмите `ПОЗВОНИТЬ`. Приложение начнет захват звука, потоковое шифрование и отправку данных на порт `9090`. На стороне сервера (Astra Linux) звук начнет воспроизводиться в реальном времени.

## Отказ от ответственности

Данный проект разработан в образовательных и исследовательских целях. Алгоритм A5/1 является историческим стандартом сетей GSM и на сегодняшний день считается криптографически уязвимым к некоторым видам атак. Не используйте его для передачи критически конфиденциальной информации в продакшене.
